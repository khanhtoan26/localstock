## Typography

Existing scale from Phase 7/9 preserved. Phase 10 defines specific role usage within the hover card:

| Role | Size | Weight | Line Height | Phase 10 Usage |
|------|------|--------|-------------|----------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 | Hover card short definition text |
| Label | 12px (`text-xs`) | 400 (regular) | 1.4 | Hover card "Xem chi tiết →" link, formula block text |
| Heading | 14px (`text-sm`) | 600 (semibold) | 1.2 | Hover card term name |

### Inline Term Typography

Glossary-linked terms inherit all typography from their surrounding prose context. The only visual differentiation is the dotted underline — no font size, weight, or color change on the text itself. This ensures linked terms flow naturally within the AI report prose.

### Hover Card Formula Block

When a glossary entry includes a `formula` field, it renders in a monospace code block:

| Property | Value |
|----------|-------|
| Font family | `font-mono` (system monospace stack) |
| Font size | 12px (`text-xs`) |
| Background | `bg-muted` |
| Padding | 8px (`p-2`) |
| Border radius | `rounded` (4px) |
| Margin top | 8px (`mt-2`) |

## Page Structure & Layout

Phase 10 does NOT introduce new pages. It enhances the existing AI report panel within the stock page (`/stock/[ticker]`).

### Glossary-Linked AI Report

**Focal point:** The AI report text — where dotted-underline terms catch the reader's eye and invite exploration via hover/tap.

```
┌─────────────────────────────────────────────────────┐
│ AI Report Panel (existing)                          │
│                                                     │
│ ## Phân Tích Kỹ Thuật                               │
│                                                     │
│ Chỉ số ·R·S·I· hiện tại ở mức 72, nằm trong       │
│ vùng quá mua. Đường ·M·A·C·D· đang có tín hiệu    │
│ phân kỳ âm...                                       │
│                                                     │
│ (· = dotted underline on first occurrence only)      │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### Hover Card Popover

**Focal point:** The term name at the top of the card — anchors the user's attention and confirms what they're looking at.

```
                    ┌────────────────────────────┐
                    │ Chỉ số sức mạnh tương đối  │ ← term (semibold)
                    │ (RSI)                       │
                    │                             │
                    │ Chỉ báo dao động đo lường   │ ← shortDef (muted)
                    │ tốc độ và biên độ thay đổi  │
                    │ giá...                      │
                    │                             │
                    │ ┌─────────────────────────┐ │
                    │ │ RSI = 100 - (100 /      │ │ ← formula (mono, bg-muted)
                    │ │ (1 + RS))               │ │    (only if formula exists)
                    │ └─────────────────────────┘ │
                    │                             │
                    │ Xem chi tiết →              │ ← link (text-primary, text-xs)
                    └────────────────────────────┘
                              ▲
                    ┈┈┈RSI┈┈┈  (dotted underline trigger)
```

- Hover card max width: 320px (`max-w-xs`) — per CONTEXT.md D-02
- Hover card padding: 16px (`p-4`)
- Hover card border: `border` (1px solid border color)
- Hover card shadow: `shadow-md`
- Hover card border radius: `rounded-lg` (8px)
- Hover card z-index: `z-50` — above all page content
- Popover renders via `Popover.Portal` — at document body level, avoiding DOM nesting issues with `<p>` elements

### Responsive Behavior

| Breakpoint | Hover Card Width | Trigger Behavior |
|------------|------------------|------------------|
| < 640px (mobile) | max-w-xs (320px), may shrink to viewport | Tap to open, tap outside to dismiss |
| ≥ 640px (desktop) | max-w-xs (320px) fixed | Hover with 200ms delay |

Hover card positioning: `@base-ui/react` Popover.Positioner handles viewport-aware positioning with automatic flip (if card would overflow viewport edge, it flips to opposite side). `sideOffset={8}` provides 8px gap between trigger and card.

## Interaction Contract

### Glossary Term Hover (Desktop)

| Property | Value |
|----------|-------|
| Trigger | Mouse enters the dotted-underline term `<span>` |
| Open delay | 200ms (`delay={200}`) — per CONTEXT.md D-09 |
| Close delay | 300ms (`closeDelay={300}`) — allows mouse travel from trigger to card |
| Hover bridge | Built-in to `@base-ui/react` Popover — card stays open while mouse moves between trigger and popup |
| Dismiss | Mouse leaves both trigger and card (after close delay), or press Escape |
| Underline transition | Dotted → solid on hover (`hover:decoration-solid`) |

### Glossary Term Tap (Mobile)

| Property | Value |
|----------|-------|
| Trigger | Single tap on the dotted-underline term |
| Dismiss | Tap outside the card, or press back |
| No long-press | Simple tap toggle — per CONTEXT.md D-10 |

### Click-Through Navigation

| Property | Value |
|----------|-------|
| Trigger | Click/tap "Xem chi tiết →" link inside hover card |
| Target | `/learn/{category}#{entry.id}` (e.g., `/learn/technical#rsi`) |
| Routing | Next.js `<Link>` for client-side navigation — per CONTEXT.md D-13 |
| Target behavior | Learn page auto-scrolls to and auto-expands matching entry — already implemented in Phase 9 |
| Card dismiss | Card closes when navigation occurs (standard behavior) |

### Term Matching Rules (Visual Behavior)

| Rule | Behavior | Source |
|------|----------|--------|
| First occurrence only | Only the first occurrence of each glossary entry is highlighted with dotted underline. Subsequent occurrences render as normal text. | CONTEXT.md D-07 |
| Longest-first matching | When aliases overlap (e.g., "chỉ số RSI" vs "RSI"), the longer alias takes priority. | CONTEXT.md D-05 |
| Case-insensitive | "rsi", "RSI", "Rsi" all match the same entry. | CONTEXT.md D-06 |
| Inline rendering | Terms are `<span>` elements that flow inline within prose. No line breaks or block-level interruptions. | CONTEXT.md D-04 |
| No code block scanning | Terms inside `<code>` or `<pre>` elements are NOT scanned for glossary matches. | RESEARCH.md A2 |

## Visual States

### Glossary Term States

| State | Visual | CSS |
|-------|--------|-----|
| Default | Dotted underline in accent color, text inherits prose color | `decoration-dotted underline underline-offset-2 decoration-primary` |
| Hover | Solid underline in accent color | `hover:decoration-solid` |
| Active/Pressed | Solid underline (same as hover) | Inherits hover state |
| Popover open | Solid underline persists while card is visible | Managed by Popover trigger active state |
| Cursor | Pointer cursor on glossary terms | `cursor-pointer` |

### Hover Card States

| State | Visual |
|-------|--------|
| Opening | Appears with default `@base-ui/react` Popover transition (agent discretion on exact animation) |
| Visible | Card with `border bg-popover p-4 shadow-md rounded-lg`, positioned above or below trigger |
| Closing | Fades out with Popover default transition |

### Hover Card Content States

| Content State | Visual |
|---------------|--------|
| With formula | Term + shortDef + formula code block + "Xem chi tiết →" link (4 sections) |
| Without formula | Term + shortDef + "Xem chi tiết →" link (3 sections) |
| Entry not found | Should not occur (glossary data is static). If it does, term renders as normal text without dotted underline — silently degrade. |

## Accessibility Contract

| Requirement | Standard | Implementation |
|-------------|----------|----------------|
| Popover trigger role | WCAG 2.1 4.1.2 | `@base-ui/react` Popover auto-manages `aria-haspopup`, `aria-expanded`, `aria-describedby` on trigger |
| Keyboard open/close | WCAG 2.1 2.1.1 | Enter/Space opens popover when trigger is focused; Escape closes it — built-in to `@base-ui/react` Popover |
| Focus management | WCAG 2.1 2.4.3 | When popover opens via keyboard, focus moves to popup content; when closed, focus returns to trigger — built-in |
| "Xem chi tiết" link | WCAG 2.1 2.4.4 | `aria-label="Xem chi tiết về {term}"` provides unique link purpose |
| Inline trigger element | WCAG 2.1 4.1.2 | `nativeButton={false}` with `render={<span />}` — renders as inline span, not block button, preserving prose flow |
| Color contrast | WCAG AA 4.5:1 | Dotted underline uses `decoration-primary` — accent color passes WCAG AA on both light and dark backgrounds (Phase 7 verified). Term text inherits prose foreground color. |
| No reliance on color alone | WCAG 2.1 1.4.1 | Terms are visually differentiated by underline style (dotted), not just color — sighted users who cannot distinguish colors can still identify linked terms |
