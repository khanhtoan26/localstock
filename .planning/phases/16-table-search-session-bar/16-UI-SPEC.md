## Spacing Scale

Standard 8-point scale. All values are Tailwind utility classes mapping to these pixel values.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px (`gap-1`, `p-1`) | Icon-to-text gap inside session bar, sort icon inline gap |
| sm | 8px (`gap-2`, `p-2`) | Search input internal padding top/bottom, session bar item gap |
| md | 16px (`px-4`, `gap-4`) | Search bar to table gap, header horizontal padding (existing) |
| lg | 24px (`mb-6`) | Page title to search bar gap (matches existing `mb-6` on h1) |
| xl | 32px | Not used in new components |
| 2xl | 48px | Empty/error state vertical padding (existing pattern: `py-20`) |
| 3xl | 64px | Page-level padding (existing `py-6 pr-6` + `pl-6/pl-[280px]`) |

Exceptions:
- Header height is fixed at `h-12` (48px) — existing constraint, not changed.
- Sort icon size: `h-3 w-3` (12px) — intentionally small to stay inline with 12px column header text.
- Search input height: `h-9` (36px) — matches existing shadcn Input component default.
- Search clear button tap target: minimum `p-1` = 8px padding around `h-4 w-4` X icon, total ~24px. Acceptable for desktop-only scope.
- Session bar progress track height: `h-1` (4px) — matches existing `Progress` component ProgressTrack default.

## Color

All colors use CSS custom property tokens only. No hardcoded hex values in new components.
Source: Phase 14 palette decisions + globals.css audit.

| Role | CSS Token | Light value | Dark value | Usage |
|------|-----------|-------------|------------|-------|
| Dominant (60%) | `var(--background)` | hsl(44 25% 94.5%) warm cream | hsl(60 3% 12%) near-black | App background, main panel |
| Secondary (30%) | `var(--card)` / `var(--muted)` | white / warm cream | hsl(60 3% 12%) / hsl(50 3% 16%) | Card surfaces, table rows, search bar container |
| Accent (10%) | `var(--accent)` | hsl(44 15% 93%) | hsl(50 3% 18%) | Hover state on table rows, search input focus ring uses `var(--ring)` |
| Destructive | `var(--destructive)` | hsl(0 56.2% 45.4%) | hsl(0 67% 60%) | Not used in this phase (no destructive actions) |

**Semantic tokens in use this phase:**

| Token | Usage |
|-------|-------|
| `var(--foreground)` | Primary text: sort icon active state, session bar text when market open |
| `var(--muted-foreground)` | Subdued text: column headers, search placeholder, session bar closed state text, sort icon inactive |
| `var(--border)` | Search input border (via Input component `border-input`), session bar progress track separator |
| `var(--muted)` | Session bar progress track background (matches existing ProgressTrack `bg-muted`) |
| `var(--primary)` | Session bar progress indicator fill (matches existing ProgressIndicator `bg-primary`) |
| `var(--ring)` | Search input focus ring (via Input component `focus-visible:ring-ring/50`) |

**Accent reserved for (explicit list):**
- Table row hover background: `hover:bg-muted/50` (existing)
- Search input focus ring: `var(--ring)` at 50% opacity (via shadcn Input, not var(--accent))
- Sidebar icon hover: `hover:bg-accent` (existing, not new)

**Accent is NOT used for:**
- Session bar fill (uses `var(--primary)`)
- Sort icons (uses `var(--muted-foreground)` inactive / `var(--foreground)` active)
- Search clear button (uses `var(--muted-foreground)` with hover `var(--foreground)`)

## Component Inventory

New components to create in this phase. Existing components are extended, not replaced.

### New: `MarketSessionBar`

**File:** `apps/helios/src/components/layout/market-session-bar.tsx`

**Directive:** `"use client"`

**Behavior:**
- Renders in the header center `flex-1` region between logo block and right controls.
- Calculates current HOSE session from `Date.now()` converted to UTC+7 (`new Date().toLocaleString("en-US", { timeZone: "Asia/Ho_Chi_Minh" })`).
- Refreshes every 60 seconds via `setInterval` in `useEffect`.
- Hidden on small screens: `hidden sm:flex`.

**Layout structure (outer → inner):**
```
<div className="hidden sm:flex items-center justify-center flex-1">
  <div className="flex items-center gap-2 text-sm text-muted-foreground">
    [phase label]  [progress track]  [countdown text]
  </div>
</div>
```

**Market open state** (Pre-market through ATC):
- Phase label: `<span className="text-xs font-medium text-foreground w-20 text-right">ATO</span>` — fixed 80px right-aligned label ensures progress bar stays centered.
- Progress track: `<div className="w-24 h-1 rounded-full bg-muted overflow-hidden"><div className="h-full bg-primary rounded-full transition-all" style={{ width: `${pct}%` }} /></div>` — 96px wide, 4px tall.
- Countdown: `<span className="text-xs text-muted-foreground w-16">8m left</span>` — fixed 64px left-aligned.
- Full example at 30% progress through ATO: `[ ATO ████░░░░░░░░ 8m left ]`

**Market closed state** (Closed phase, weekends, pre-8:30):
- No phase label shown.
- Progress bar shows 0% fill.
- Single centered text: `<span className="text-xs text-muted-foreground">● Closed · Opens in 14h 30m</span>`
- The `●` dot uses a literal Unicode bullet, not a SVG icon.

**HOSE Phase boundaries (UTC+7):**

| Phase label | Start (UTC+7) | End (UTC+7) | Display label |
|-------------|---------------|-------------|---------------|
| Pre-market | 08:30 | 09:00 | Pre-market |
| ATO | 09:00 | 09:15 | ATO |
| Morning | 09:15 | 11:30 | Morning |
| Lunch | 11:30 | 13:00 | Lunch |
| Afternoon | 13:00 | 14:30 | Afternoon |
| ATC | 14:30 | 14:45 | ATC |
| Closed | 14:45+ / weekends | — | (closed state) |

Weekday check: `getDay() === 0` (Sunday) or `getDay() === 6` (Saturday) → closed state.

Countdown format during active phase: `{N}h {M}m left` if ≥60 minutes remain; `{M}m left` if <60 minutes remain.

Countdown format when closed: `Opens in {N}h {M}m` — always show hours even if 0 (e.g., `Opens in 0h 45m`).

Add a `// TODO: Add Vietnamese public holiday awareness (Tết, 30/4, 2/9) in future` comment directly in the component above the weekday check logic.

### New: `StockSearchInput`

**File:** `apps/helios/src/components/rankings/stock-search-input.tsx`

**Directive:** `"use client"` (uses nuqs `useQueryState`)

**Layout placement:** Between `<h1>` and `<StockTable>` in `rankings/page.tsx`. Wrapped in `<div className="mb-4">`.

**Visual specification:**

```
[ Search icon ][ "Filter by symbol or name..." ][ ×clear ]
```

- Outer wrapper: `<div className="relative max-w-xs">` — 320px max width, left-aligned.
- Search icon: `<Search className="absolute left-2.5 top-2.5 h-4 w-4 text-muted-foreground pointer-events-none" />` — 10px from left edge, centered vertically.
- Input: shadcn `<Input>` with `className="pl-8 pr-8 text-sm h-9"` — left padding accommodates icon, right padding accommodates clear button.
- Placeholder: `"Filter by symbol or name..."` (EN) / `"Lọc theo mã hoặc tên..."` (VI) — add to message files.
- Clear button: visible only when `q` is non-empty. `<button>` with `<X className="h-4 w-4" />` positioned `absolute right-2 top-2.5 text-muted-foreground hover:text-foreground transition-colors`. No label text, `aria-label="Clear search"`.

**State:** `const [q, setQ] = useQueryState("q", { defaultValue: "", shallow: true })` via nuqs.

**Debounce:** 150ms debounce on input change before calling `setQ`. Use local `useState` for immediate input display, debounced effect for URL update.

**Filter logic (in `rankings/page.tsx`, not in StockSearchInput):**

```ts
const filtered = useMemo(() => {
  if (!q.trim()) return data.stocks;
  const lower = q.toLowerCase();
  return data.stocks.filter(
    (s) =>
      s.symbol.toLowerCase().startsWith(lower) ||
      (s.name ?? "").toLowerCase().includes(lower)
  );
}, [data.stocks, q]);
```

Note: `StockScore` currently has no `name` field — symbol prefix match is the primary filter. The `name` branch is written defensively for when the type is extended.

**Empty search result state:** When filtered array is empty, render:
```
<p className="text-sm text-muted-foreground py-8 text-center">
  No stocks match "{q}". Try a different symbol.
</p>
```
(Vietnamese: `Không tìm thấy mã "{q}". Thử tìm mã khác.`)

## Interaction Contracts

### Table Sort

| Interaction | Behavior |
|-------------|----------|
| Click sortable column header | Toggle sort direction if same column; default to desc if new column |
| Click Grade column header | Sort by `GRADE_RANK` map value; desc shows A+ first (rank 1 = lowest value = top) |
| Click Recommendation header | No-op (non-sortable). Cursor remains default, no icon shown |
| Equal values | Secondary sort by `symbol.localeCompare(b.symbol)` ascending, always |
| Null score values | `-Infinity` sentinel keeps null rows at bottom in desc (existing behavior, unchanged) |
| Initial state | Sort by `total_score` desc (existing default, unchanged) |

### Search Input

| Interaction | Behavior |
|-------------|----------|
| Type in search | 150ms debounce, then URL param `?q=` updates, table filters immediately in browser |
| Click × clear | Sets `q` to `""`, input clears, URL param removed, full table shown |
| Press Escape | Blur input only (do NOT clear search — user may want to return to search) |
| Navigate away and back | URL param `?q=` preserved in browser history; nuqs restores filter on mount |
| Empty query | Shows all 50 stocks (no filter applied) |
| No results | Inline empty message (see Copywriting below) |

### Session Bar

| Interaction | Behavior |
|-------------|----------|
| Component mounts | Calculates initial state synchronously, renders immediately (no loading flash) |
| Every 60 seconds | `setInterval` fires, recalculates phase and countdown, re-renders |
| Phase boundary crossed | Next tick recalculates — user may see up to 60s delay at exact boundary (acceptable) |
| Weekend / holiday | Shows closed state with countdown to Monday 08:30 UTC+7 |
| Small screens (<sm breakpoint) | Component hidden via `hidden sm:flex` — header collapses to just logo + controls |
