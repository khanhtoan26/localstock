### Progress Stepper Design

For single stock generation, 3 steps:
1. **Queued** → pending state
2. **Generating** → running state (shows streaming text or spinner)
3. **Complete** → completed state (transitions to report preview)

For batch (N stocks), the sheet shows:
- Current stock being generated (symbol + step indicator)
- Overall batch progress bar: `[████░░░░] 2/5 stocks`
- Completed stocks listed below with ✓ checkmarks

### Step Indicator Styling

| State | Dot | Label | Sublabel |
|-------|-----|-------|----------|
| Pending | `○` 8px border-muted-foreground | text-muted-foreground | — |
| Active | `●` 8px bg-primary, pulse animation | text-foreground font-medium | Streaming text in text-muted-foreground font-mono |
| Completed | `✓` 8px bg-stock-up text-white | text-muted-foreground line-through | — |
| Failed | `✗` 8px bg-destructive text-white | text-destructive | Error message |

## Interaction Contract

### Flow: Single Stock Report Generation

1. User selects 1 stock in Pipeline tab → clicks "Report" button
2. `useTriggerAdminReport` mutation fires → toast: "Report operation started"
3. Sheet opens automatically with symbol in header
4. Progress stepper shows Queued → Generating → Complete
5. On job completion (detected by `useJobTransitions`), sheet transitions to report preview
6. Report preview renders `AIReportPanel` with fetched report data via `useStockReport(symbol)`
7. User reads report in-sheet or clicks "View Stock Page" to navigate

### Flow: Batch Report Generation

1. User selects N stocks → clicks "Report"
2. Backend creates N sequential jobs (per D-04)
3. Sheet opens, shows batch progress: "Generating reports: 1/N"
4. Each stock transitions through steps; completed stocks stack above
5. On all jobs complete, sheet shows last generated report with batch summary
6. Progress bar: `<Progress value={(completed/total)*100} />`

### Flow: Error Recovery

1. Job transitions to "failed" status
2. Sheet shows error state with AlertCircle icon + error message
3. "Retry" button re-triggers `useTriggerAdminReport` for failed symbol(s)
4. "Close" button dismisses sheet

### Data Fetching Strategy

- **Job status polling:** Existing `useAdminJobs` already polls at 3s when active jobs exist — reuse this
- **Job transition detection:** `useJobTransitions` hook fires callback on status change — use to update sheet state
- **Report data:** After job completes, fetch via `useStockReport(symbol)` — cache already invalidated by `invalidateForJob`
- **No new SSE/WebSocket needed:** Polling-based approach is sufficient since Ollama generation takes 30-120s per stock; 3s polling interval is acceptable latency

### Sheet Open/Close Rules

| Trigger | Action |
|---------|--------|
| Report button clicked | Open sheet immediately |
| Report job completed (toast) | Sheet stays open if already open; toast "View job" action focuses job in Jobs tab (existing behavior) |
| All batch jobs complete | Sheet stays open showing final report |
| User clicks X / overlay | Close sheet, no confirmation needed (non-destructive) |
| Escape key | Close sheet |
| Report already generating (sheet closed) | User can reopen from a toast action or by clicking Report again |

## State Machine

```
Sheet States:
  CLOSED → GENERATING → COMPLETED
                      → FAILED → GENERATING (retry)
                               → CLOSED

GENERATING sub-states (per stock):
  QUEUED → RUNNING → DONE
                   → ERROR

Batch tracking:
  { currentIndex: number, total: number, results: Map<symbol, 'done'|'error'> }
```

## Design Decisions Summary

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Sheet vs Dialog | Sheet (right drawer) | Matches existing stock page drawer pattern (Phase 8 STOCK-03). Provides more vertical space for report content. Non-blocking — user can still see admin page behind overlay. |
| Polling vs SSE | Polling (3s via existing useAdminJobs) | No new backend endpoint needed. 3s polling is acceptable for 30-120s generation time. Existing `useJobTransitions` hook handles state change detection. Simplest implementation. |
| Progress UI | Step indicator + batch progress bar | Steps show clear lifecycle (queued→generating→complete). No streaming text preview — backend doesn't expose token-level streaming, only job status. Progress bar for batch gives clear "X of Y" feedback. |
| Report rendering | Reuse AIReportPanel | Consistent with stock detail page. Already handles recommendation badge, grade badge, score, T+3, markdown via GlossaryMarkdown. No duplication. |
| Batch display | One sheet, sequential progress | Per D-04: sequential generation. Sheet shows current stock + overall batch counter. Completed stocks don't need individual preview — user can view from stock page. |
