### File 1: `apps/helios/src/components/admin/report-progress.tsx`

Presentational component rendering a 3-step indicator (Queued → Generating → Complete) for a report job.

```typescript
"use client";

import { useTranslations } from "next-intl";
import { Check, Loader2, AlertCircle } from "lucide-react";
import { cn } from "@/lib/utils";
```

**Props interface:**
```typescript
interface ReportProgressProps {
  jobStatus: "pending" | "running" | "completed" | "failed";
  currentSymbol?: string;
}
```

**Implementation details:**
- Define 3 steps: "stepQueued" (pending), "stepGenerating" (running), "stepComplete" (completed)
- Map `jobStatus` to step states:
  - `pending`: step 1 = active, steps 2-3 = pending
  - `running`: step 1 = completed, step 2 = active, step 3 = pending
  - `completed`: all steps = completed
  - `failed`: step 1 = completed (if was running), step 2 = failed, step 3 = pending
- Each step renders a dot + label:
  - Pending dot: `"h-2 w-2 rounded-full border border-muted-foreground"` + label `"text-sm text-muted-foreground"`
  - Active dot: `"h-2 w-2 rounded-full bg-primary step-active-pulse"` + label `"text-sm text-foreground font-medium"`
  - Completed dot: render `<Check className="h-3 w-3 text-[var(--stock-up)]" />` instead of dot + label `"text-sm text-muted-foreground line-through"`
  - Failed dot: render `<AlertCircle className="h-3 w-3 text-destructive" />` instead of dot + label `"text-sm text-destructive"`
- When `jobStatus === "running"` and `currentSymbol` is provided, show below the active step: `<span className="text-xs text-muted-foreground ml-4">{t("generatingFor", { symbol: currentSymbol })}</span>`
- Add `aria-live="polite"` on the outer container for screen reader accessibility
- Wrap in `<div className="space-y-3">` container
- Export as named: `export function ReportProgress(...)`
