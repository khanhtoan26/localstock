```tsx
const mainNavItems = [
  { href: "/rankings", labelKey: "rankings" as const, icon: BarChart3 },

  { href: "/learn", labelKey: "learn" as const, icon: BookOpen },
];

  { href: "/admin", labelKey: "admin" as const, icon: Shield },
];
```

**Component structure** — `export function FloatingSidebar()`:

1. Use hooks: `usePathname()`, `useTranslations("nav")`, `useSidebarState()`.
2. Define `handleNavClick` — when expanded, clicking a nav link auto-collapses the sidebar:
   ```tsx
   const handleNavClick = () => { if (!collapsed) toggle(); };
   ```
3. Render a React fragment containing two `<aside>` elements:

**Part A: Icon Rail** (always visible, per D-01, D-02, D-03):
- `<aside>` with classes: `fixed left-0 top-0 h-screen w-14 z-30 flex flex-col items-center py-3 border-r border-sidebar-border bg-sidebar`
- Contains a `<nav>` for Main group icons at top — each icon wrapped in `<Tooltip>`:
  - `<TooltipTrigger delay={200} render={<Link href={href} onClick={collapsed ? toggle : handleNavClick} className={cn("flex items-center justify-center w-10 h-10 rounded-md", active ? "bg-sidebar-accent text-sidebar-primary" : "text-sidebar-foreground/70 hover:bg-sidebar-accent hover:text-sidebar-foreground")} />}>`
  - Icon component inside trigger: `<Icon className="h-5 w-5" />`
  - Tooltip portal rendered conditionally ONLY when `collapsed` is true (per Pitfall 3):
    ```tsx
    {collapsed && (
      <TooltipPortal>
        <TooltipPositioner side="right" sideOffset={8}>
          <TooltipContent>{t(labelKey)}</TooltipContent>
        </TooltipPositioner>
      </TooltipPortal>
    )}
    ```
  - Active state detection: `pathname.startsWith(href)`
- `<div className="flex-1" />` spacer between Main and Admin groups
- `<div className="w-8 border-t border-sidebar-border mb-2" />` separator line
- `<nav>` for Admin group icons at bottom (same tooltip pattern, with `mb-2`)

**Part B: Overlay Panel** (always in DOM, visibility via CSS transform, per D-06, D-07, D-08):
- `<aside>` with classes:
  ```
  fixed left-14 top-0 h-screen w-60 z-40
  border-r border-sidebar-border bg-sidebar shadow-lg
  transition-transform duration-[180ms] ease-out
  ```
  Plus conditional: `collapsed ? "-translate-x-full" : "translate-x-0"`
- Add `aria-hidden={collapsed}` for accessibility (Pitfall 7)
- Contains:
  - Header div (per D-02 — only in expanded): `<h1 className="text-lg font-bold text-sidebar-primary">LocalStock</h1>` and `<p className="text-xs text-muted-foreground">AI Stock Agent</p>` inside `<div className="p-4 border-b border-sidebar-border">`
  - Full nav with labels: map over `[...mainNavItems, ...adminNavItems]` rendering `<Link>` elements with:
    - Classes: `flex items-center gap-3 px-3 py-2 rounded-md text-sm`
    - Active state: `bg-sidebar-accent text-sidebar-primary`
    - Inactive state: `text-sidebar-foreground/70 hover:bg-sidebar-accent`
    - `onClick={handleNavClick}` (auto-collapse after navigation)
    - Icon (`h-4 w-4`) + `{t(labelKey)}` text label

**Critical implementation notes:**
- Do NOT conditionally render the overlay panel (`{!collapsed && ...}`). ALWAYS render it and use CSS `transform` to hide — this enables the slide animation (Pitfall 2).
- Do NOT add a backdrop overlay (D-07 explicitly rejects this).
- Do NOT add badge indicators on icons (D-03).
- Do NOT show LocalStock header/branding in the icon rail (D-02).
- Tooltips ONLY appear when collapsed — when expanded, labels are visible in the panel (Pitfall 3).
