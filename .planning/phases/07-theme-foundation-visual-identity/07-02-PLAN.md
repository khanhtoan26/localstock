**Step 3 — Create `useChartTheme` hook (per D-12):**

Create `apps/helios/src/hooks/use-chart-theme.ts`:

```typescript
"use client";
import { useTheme } from "next-themes";
import { getChartColors, type ChartColorSet } from "@/lib/chart-colors";

/** Returns the chart color set for the current theme */
export function useChartTheme(): ChartColorSet {
  const { resolvedTheme } = useTheme();
  // resolvedTheme can be undefined during SSR — default to light (warm-light is default)
  return getChartColors(resolvedTheme === "dark" ? "dark" : "light");
}
```

Follow codebase patterns:
- `"use client"` as first line — required for useTheme hook
- Named export function with `use` prefix — matches queries.ts hook naming
- Returns data directly (the ChartColorSet) — simple, no wrapper object
- Defaults to light when resolvedTheme is undefined (SSR safety, matches D-03 default)

**Critical design per D-11:** Do NOT destroy/recreate charts on theme change. Use `chart.applyOptions()` + `series.applyOptions()` to update colors in-place. This preserves zoom/scroll state.

**Implementation strategy:** Both chart components currently have a single `useEffect` that creates the chart and sets data. The approach is:
1. Import `useChartTheme` instead of `CHART_COLORS`
2. Use `chartColors` from the hook for initial chart creation colors
3. Store series refs so we can call `series.applyOptions()` on theme change
4. Add a SECOND `useEffect` that depends on `chartColors` and calls `applyOptions` on the stored refs
5. Include `chartColors` in the creation effect's deps so initial render uses correct theme

Keep `chartColors` OUT of the creation effect's dependency array. Instead, add a second useEffect:

```typescript
// Effect 1: Create chart (NO chartColors in deps — only runs on data change)
useEffect(() => {
  if (!containerRef.current || !prices.length) return;
  const chart = createChart(containerRef.current, {
    layout: {
      background: { color: chartColors.chartBg },
      textColor: chartColors.chartText,
    },
    // ... (same as current, using chartColors for initial colors)
  });
  chartRef.current = chart;

  const candleSeries = chart.addSeries(CandlestickSeries, {
    upColor: chartColors.candleUp,
    // ... other candle options
  });
  candleRef.current = candleSeries;
  candleSeries.setData(/* ... */);

  const volumeSeries = chart.addSeries(HistogramSeries, {
    priceFormat: { type: "volume" },
    priceScaleId: "volume",
  });
  volumeRef.current = volumeSeries;
  volumeSeries.setData(
    prices.map((p) => ({
      time: p.time,
      value: p.volume,
      color: p.close >= p.open ? chartColors.volumeUp : chartColors.volumeDown,
    }))
  );

  // ... indicator overlays same as current but with chartColors ...
  // ... resize observer ...

  return () => {
    resizeObserver.disconnect();
    chart.remove();
    chartRef.current = null;
    candleRef.current = null;
    volumeRef.current = null;
  };
}, [prices, indicators]); // NO chartColors here

// Effect 2: Re-theme on toggle (only runs when chartColors changes)
useEffect(() => {
  const chart = chartRef.current;
  if (!chart) return;

  chart.applyOptions({
    layout: {
      background: { color: chartColors.chartBg },
      textColor: chartColors.chartText,
    },
    grid: {
      vertLines: { color: chartColors.chartGrid },
      horzLines: { color: chartColors.chartGrid },
    },
  });

  if (candleRef.current) {
    candleRef.current.applyOptions({
      upColor: chartColors.candleUp,
      downColor: chartColors.candleDown,
      borderUpColor: chartColors.candleUp,
      borderDownColor: chartColors.candleDown,
      wickUpColor: chartColors.candleUp,
      wickDownColor: chartColors.candleDown,
    });
  }

  // Volume histogram: setData is needed to update per-bar colors
  if (volumeRef.current && prices.length) {
    volumeRef.current.setData(
      prices.map((p) => ({
        time: p.time,
        value: p.volume,
        color: p.close >= p.open ? chartColors.volumeUp : chartColors.volumeDown,
      }))
    );
  }
}, [chartColors, prices]);
```

**IMPORTANT: Volume bars require `setData` not just `applyOptions`** because volume bar colors are set per-data-point (the `color` field on each bar), not as series-level options. This is a lightweight-charts constraint — histogram bars have individual colors based on up/down direction.

```typescript
useEffect(() => {
  const chart = chartRef.current;
  if (!chart) return;

  chart.applyOptions({
    layout: {
      background: { color: chartColors.chartBg },
      textColor: chartColors.chartText,
    },
    grid: {
      vertLines: { color: chartColors.chartGrid },
      horzLines: { color: chartColors.chartGrid },
    },
  });

  if (type === "macd") {
    // MACD histogram needs setData for per-bar colors
    if (seriesRefs.current.hist && indicators.length) {
      const histData = indicators
        .filter((d) => d.macd_histogram != null)
        .map((d) => ({
          time: d.time,
          value: d.macd_histogram!,
          color: d.macd_histogram! >= 0
            ? chartColors.macdHistPositive
            : chartColors.macdHistNegative,
        }));
      seriesRefs.current.hist.setData(histData);
    }
    if (seriesRefs.current.line1) {
      seriesRefs.current.line1.applyOptions({ color: chartColors.macdLine });
    }
    if (seriesRefs.current.line2) {
      seriesRefs.current.line2.applyOptions({ color: chartColors.macdSignal });
    }
  }

  if (type === "rsi") {
    if (seriesRefs.current.line1) {
      seriesRefs.current.line1.applyOptions({ color: chartColors.rsiLine });
    }
    // Price lines (overbought/oversold) are created with createPriceLine
    // and cannot be updated via applyOptions — they use the initial colors.
    // This is acceptable since the line color (red/green) is semantically
    // the same in both themes (just different shades) and the lines are
    // decorative (thresholds at 70/30).
  }
}, [chartColors, type, indicators]);
```

In the creation effect, store series refs:
- For MACD: `seriesRefs.current.hist = histSeries`, `.line1 = macdLine`, `.line2 = signalLine`
- For RSI: `seriesRefs.current.line1 = rsiSeries`

In cleanup, reset: `seriesRefs.current = { hist: null, line1: null, line2: null }`

For RSI price lines (overbought at 70, oversold at 30), use `chartColors.rsiOverbought` and `chartColors.rsiOversold` in the creation effect for initial colors. Price lines created with `createPriceLine` don't have a public `applyOptions` — they'd need to be removed and recreated. Since these are simple threshold lines with semantically similar colors across themes, accept this limitation.
