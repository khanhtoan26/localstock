## Color

### Theme Architecture

This phase establishes a dual-theme system. Colors are specified per-theme.

| Property | Value |
|----------|-------|
| Default theme | warm-light ("claude") — maps to `:root` (no CSS class) |
| Alternate theme | dark — maps to `.dark` CSS class |
| Color space (warm-light) | oklch |
| Color space (dark) | hsl (existing, minimize diff) |
| Switching mechanism | next-themes ^0.4.6, `attribute="class"`, `enableSystem={false}` |
| Persistence | localStorage key `localstock-theme` |

### Warm-Light Theme (`:root` — DEFAULT)

| Role | oklch Value | Approx Hex | Usage |
|------|-------------|------------|-------|
| Dominant (60%) — background | `oklch(0.97 0.02 70)` | #F5F0E8 | Page background, main content area |
| Secondary (30%) — card | `oklch(0.99 0.01 70)` | #FDFBF7 | Cards, popovers, elevated surfaces |
| Secondary (30%) — sidebar | `oklch(0.96 0.015 70)` | #F0EBE3 | Sidebar background (slightly darker than main) |
| Foreground | `oklch(0.22 0.02 60)` | #2C2722 | Body text, headings |
| Muted foreground | `oklch(0.50 0.02 60)` | #7A7570 | Secondary text, labels, timestamps |
| Accent (10%) — primary | `oklch(0.70 0.14 45)` | #D97757 | Primary buttons (bg), active nav item, focus ring |
| Primary foreground | `oklch(0.99 0.01 70)` | #FDFBF7 | Text on primary buttons |
| Border | `oklch(0.88 0.02 70)` | #DDD6CB | Borders, dividers, input outlines |
| Muted/secondary bg | `oklch(0.93 0.02 70)` | #E8E2D8 | Hover states, secondary button bg |
| Destructive | `oklch(0.577 0.245 27)` | #DC2626 | Delete actions, critical errors |

Accent reserved for: primary action buttons (background), active sidebar nav link, focus ring outline, theme toggle hover state. **Never** for body text (fails WCAG AA on cream at 3.0-4.0:1 — see Pitfall 4).

## Interaction Contract

> **Focal point:** Primary focal point remains the stock rankings table/chart area — the theme toggle is a peripheral control, not a focal element.

### Theme Toggle

| Property | Value |
|----------|-------|
| Placement | AppShell header, right-aligned, inside new `<header>` strip above `<main>` |
| Component | `<Button variant="ghost" size="icon">` wrapping lucide Sun/Moon icon |
| Icon size | `h-4 w-4` (16×16px) |
| Light mode icon | Moon (click to switch to dark) |
| Dark mode icon | Sun (click to switch to light) |
| Animation on switch | Instant (`disableTransitionOnChange` on ThemeProvider) |
| SSR placeholder | Render button shell with no icon until `resolvedTheme` is defined (avoids hydration mismatch) |
| Keyboard accessible | Inherits from Button component — focusable, Enter/Space activates |

### Chart Re-theming

| Property | Value |
|----------|-------|
| Trigger | `resolvedTheme` change via `useChartTheme()` hook |
| Method | `chart.applyOptions()` + `series.applyOptions()` — NO chart destroy/recreate |
| Preserved state | Zoom level, scroll position, crosshair position |
| Latency target | Colors update within same animation frame (synchronous applyOptions) |

### FOUC Prevention

| Property | Value |
|----------|-------|
| Mechanism | next-themes inline blocking `<script>` injected before hydration |
| Hydration fix | `suppressHydrationWarning` on `<html>` element only |
| Test method | Hard refresh 10× on Slow 3G throttle — no cream→dark or dark→cream flash visible |

## Accessibility Contract
