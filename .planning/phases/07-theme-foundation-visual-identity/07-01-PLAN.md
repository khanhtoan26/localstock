**Step 2 — Create `src/components/theme/` directory and ThemeProvider:**

Create `apps/helios/src/components/theme/theme-provider.tsx` — a "use client" component wrapping next-themes' ThemeProvider with the app's configuration per D-05 and D-06:

```tsx
"use client";
import { ThemeProvider as NextThemesProvider } from "next-themes";

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  return (
    <NextThemesProvider
      attribute="class"
      defaultTheme="claude"
      themes={["claude", "dark"]}
      value={{ claude: "", dark: "dark" }}
      enableSystem={false}
      disableTransitionOnChange
      storageKey="localstock-theme"
    >
      {children}
    </NextThemesProvider>
  );
}
```

Key config per locked decisions:
- `attribute="class"` — Tailwind v4 `@custom-variant dark` uses class-based detection (D-05)
- `defaultTheme="claude"` — warm-light is default, not dark (D-03, D-05)
- `value={{ claude: "", dark: "dark" }}` — "claude" theme maps to NO class (`:root` applies), "dark" → `.dark` class (D-05)
- `enableSystem={false}` — no OS detection, warm-light forced for all new visitors (D-05)
- `disableTransitionOnChange` — prevents flash during toggle
- `storageKey="localstock-theme"` — scoped localStorage key (D-06)

**Step 3 — Create ThemeToggle component:**

Create `apps/helios/src/components/theme/theme-toggle.tsx` per D-04:

```tsx
"use client";
import { useTheme } from "next-themes";
import { Sun, Moon } from "lucide-react";
import { Button } from "@/components/ui/button";

export function ThemeToggle() {
  const { resolvedTheme, setTheme } = useTheme();

  return (
    <Button
      variant="ghost"
      size="icon"
      onClick={() => setTheme(resolvedTheme === "dark" ? "claude" : "dark")}
      aria-label={
        resolvedTheme === "dark"
          ? "Switch to light theme"
          : "Switch to dark theme"
      }
    >
      {resolvedTheme === "dark" ? (
        <Sun className="h-4 w-4" />
      ) : (
        <Moon className="h-4 w-4" />
      )}
    </Button>
  );
}
```

Follow codebase patterns:
- Named export (NOT default export) — matches all existing components
- `"use client"` as first line — required for useTheme hook
- lucide-react icons imported as named imports with `className="h-4 w-4"` sizing — matches sidebar.tsx pattern
- `Button` imported from `@/components/ui/button` — existing component
- `variant="ghost"` + `size="icon"` — existing button variants per 07-PATTERNS.md
- `resolvedTheme` used (not `theme`) — correctly resolves the actual applied theme
- Dynamic aria-label per UI-SPEC.md accessibility contract

**Step 1 — Update `layout.tsx` (per D-03, D-05, D-06):**

Three changes to `apps/helios/src/app/layout.tsx`:

1. Add import for ThemeProvider:
```tsx
import { ThemeProvider } from "@/components/theme/theme-provider";
```

2. Remove `className="dark"` from `<html>` tag and add `suppressHydrationWarning`:
```tsx
<html lang="vi" suppressHydrationWarning>
```
The `suppressHydrationWarning` is required because next-themes modifies the `<html>` className before React hydrates — without it, React throws a hydration mismatch error. This is the documented, officially blessed escape hatch for next-themes (see Pitfall 5 in RESEARCH.md).

3. Wrap `QueryProvider` with `ThemeProvider` (ThemeProvider must be outermost body child):
```tsx
<body>
  <ThemeProvider>
    <QueryProvider>
      <AppShell>{children}</AppShell>
    </QueryProvider>
  </ThemeProvider>
</body>
```
