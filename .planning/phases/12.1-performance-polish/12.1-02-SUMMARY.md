## Accomplishments

- AdminPage detects job transitions via useJobTransitions and fires toast.success/toast.error with i18n messages including job type and affected symbols
- Toast action button ("View job") switches to Jobs tab and highlights the relevant row with smooth scroll
- JobMonitor accepts focusedJobId prop, maintains callback ref Map for all job rows, applies job-row-highlight CSS animation (2300ms) and scrolls into view
- Jobs TabsContent uses keepMounted to prevent DOM unmounting, ensuring row refs persist across tab switches
- Targeted cache invalidation (invalidateForJob) runs on every transition before toast display

## Decisions Made

- **keepMounted on Jobs TabsContent:** Required to prevent unmounting when user is on another tab — ensures rowRefs.current is populated when toast action triggers tab switch + scrollIntoView
- **Toast action sets both activeTab and focusedJobId:** React batches these state updates, so JobMonitor sees the new focusedJobId on the same render cycle the tab switches
- **2300ms setTimeout matches CSS animation:** The cleanup timer exactly matches the `job-highlight` keyframe duration from globals.css (Plan 01), ensuring class removal and onFocusHandled fire after animation completes
- **Callback ref with Map<number, HTMLTableRowElement>:** Enables O(1) lookup by job.id for scrollIntoView — works with dynamic sorted/filtered lists since refs update on each render
