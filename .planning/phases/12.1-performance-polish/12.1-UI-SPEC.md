## Interaction Contracts

### 1. Toast on Job Completion/Failure

**Trigger:** When a polled job transitions to `completed` or `failed` (D-02), show a sonner toast.

**Toast type mapping:**

| Job status | Sonner method | Icon (auto from sonner.tsx) | Auto-dismiss |
|------------|---------------|----------------------------|--------------|
| `completed` | `toast.success()` | `CircleCheckIcon` (green) | 5 seconds |
| `failed` | `toast.error()` | `OctagonXIcon` (red) | 8 seconds (longer for errors) |

**Toast action:** Each toast includes an action button that switches the active tab to "jobs" and scrolls/highlights the relevant job row.

**Toast position:** Bottom-right (sonner default — already established).

### 2. Auto-Focus / Row Highlight in Jobs Tab

**Trigger:** When a new job is created (after trigger) or an existing job transitions to `completed`/`failed`.

**Behavior sequence:**
1. If the Jobs tab is not active, the toast action button switches to it.
2. The target job row receives the highlight animation (primary-8% background fade).
3. The row is scrolled into view via `scrollIntoView({ behavior: "smooth", block: "nearest" })`.
4. Highlight fades out over 2000ms.

**CSS class for highlight:**
```
.job-row-highlight {
  animation: job-highlight 2300ms ease-out forwards;
}

@keyframes job-highlight {
  0%   { background-color: color-mix(in srgb, var(--primary) 8%, transparent); }
  13%  { background-color: color-mix(in srgb, var(--primary) 8%, transparent); }
  100% { background-color: transparent; }
}
```

Duration breakdown: 300ms hold at 8% opacity → 2000ms fade to transparent.

### 3. Tab Auto-Switch on Job Trigger

**Existing behavior (preserved):** `PipelineControl` already calls `onOperationTriggered(() => setActiveTab("jobs"))` to switch to Jobs tab after triggering an operation. This phase does NOT change this — it layers toast notifications on top.

### 4. Cache Invalidation Feedback

**Behavior:** When a job completes, targeted query invalidation runs silently — no additional UI feedback beyond the toast. The stock table, pipeline control, and job monitor reactively update via TanStack Query's built-in refetch. Users see fresh data the next time they navigate to the relevant tab.

**No loading indicator during invalidation** — the invalidation is a background refetch that TanStack Query handles transparently. If users are on the affected tab, data updates in-place.

## Accessibility Notes

- **Toast auto-dismiss:** Success toasts dismiss at 5s, error toasts at 8s. Sonner already uses `role="status"` and `aria-live="polite"` for screen readers.
- **Row highlight animation:** Purely decorative (uses `background-color` animation). Does not affect focus management. The `scrollIntoView` provides the actual functional focus.
- **No new keyboard interactions:** This phase does not add keyboard shortcuts. Tab switching via toast action uses existing `setActiveTab` which updates visible content reactively.

---

## Performance Notes

- **Table rendering optimization** is at the agent's discretion (D-agent-discretion). If implemented, changes must stay within the existing client-side memoized filter/sort + 50-row pagination pattern. No virtual scrolling needed for ~400 symbols.
- **Polling-driven updates:** Job status changes are detected via the existing `useAdminJobs` polling query. No WebSocket or SSE additions.
