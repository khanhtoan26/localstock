<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| call-site → log sink | Sensitive values may be passed in `extra` or interpolated; redaction is the last gate before serialization |
| stdlib → loguru | Library code (uvicorn, sqlalchemy) emits via stdlib; InterceptHandler is the bridge |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01 | Information Disclosure | logging.py prod sink | mitigate | `diagnose=False` on prod sink (Pitfall 17); patcher redacts `_SENSITIVE_KEYS` and URL credentials |
| T-22-02 | Tampering | InterceptHandler frame-walking | accept | stdlib emitters trusted; depth calculation copied from loguru official recipe |
| T-22-03 | Denial of Service | enqueue=True worker thread | mitigate | Detect PYTEST_CURRENT_TEST → enqueue=False; conftest finalizer calls `logger.complete()` to drain (Pitfall 16, D-08) |
| T-22-04 | Information Disclosure | record["extra"] dict | mitigate | Patcher iterates extra keys case-insensitively against deny-list; replaces with `***REDACTED***` |
</threat_model>
