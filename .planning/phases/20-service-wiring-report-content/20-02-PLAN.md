<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM output → StockReport | LLM may hallucinate prices — mitigated by pre-computing and overwriting |
| News article titles → prompt | Article text could attempt prompt injection — mitigated by using titles only (≤500 chars) and limiting to 3 |
| DB data → computation | None values from empty/new stocks could cause arithmetic errors |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-03 | Tampering | LLM price output | mitigate | Pre-compute entry/SL/TP in Python; overwrite report fields before validation; _validate_price_levels() rejects out-of-range |
| T-20-04 | Tampering | News titles in prompt | accept | Titles are ≤500 chars from DB, limited to 3; no user-controlled content; sentiment classification already truncates to 2000 chars |
| T-20-05 | Information Disclosure | Score delta | accept | Internal data only; no PII; delta is a numeric difference |
| T-20-06 | Denial of Service | Catalyst news query | mitigate | Limit to 5 sentiment scores and 3 article titles per stock; per-stock error isolation via try/except |
</threat_model>
