| Role | Token | Light Value | Dark Value | Usage |
|------|-------|-------------|------------|-------|

| Secondary (30%) | `--card` | hsl(0 0% 100%) | hsl(60 2.1% 18.4%) | Tab content areas, table container |

### Status Badge Color Map

Job/operation status indicators use Badge component variants + custom semantic classes:

| Status | Badge Variant | Color Token | Light Rendered | Dark Rendered |
|--------|---------------|-------------|----------------|---------------|
| completed | custom class | `--stock-up` | hsl(142 72% 29%) — green | #22c55e — green |
| failed | `destructive` | `--destructive` | hsl(0 56.2% 45.4%) — red | hsl(0 67% 59.6%) — red |
| running | custom class | `--stock-warning` | hsl(48 96% 40%) — amber | #eab308 — yellow |
| pending | `secondary` | `--secondary` | hsl(53 28.6% 94.5%) — muted | hsl(30 3.3% 11.8%) — muted |

Implementation: `completed` and `running` badges use `className` override on the Badge component with `bg-[hsl(var(--stock-up))]/10 text-[hsl(var(--stock-up))]` and `bg-[hsl(var(--stock-warning))]/10 text-[hsl(var(--stock-warning))]` respectively (matching the `destructive` variant pattern of 10% background + full text color).

## Interaction Contract

### Stock Add Flow (Stocks Tab)
1. User types symbol in Input field (placeholder: "VNM, FPT, HPG...")
2. Input validates on submit: uppercase alphanumeric only (`^[A-Z0-9]+$`)
3. Invalid input: show `aria-invalid` state on Input (red border via existing styles)
4. Submit via "Add Stock" Button (variant="default", size="default") or Enter key
5. On success: toast "Stock {SYMBOL} added" + refetch stock list + clear input
6. On 409 conflict: toast "Stock {SYMBOL} already tracked" (warning tone)
7. On error: toast "Failed to add stock" with error detail

### Stock Remove Flow (Stocks Tab)
1. Each row has icon-only Button (variant="ghost", size="icon-xs") with X icon
2. Click removes immediately (no modal — action is reversible by re-adding)
3. On success: toast "Stock {SYMBOL} removed" + refetch stock list
4. On error: toast "Failed to remove stock" with error detail

### Pipeline Trigger Flow (Pipeline Tab)
1. Stock table shows checkboxes (first column) for row selection
2. Header checkbox toggles select all / deselect all
3. Action buttons in bar above table:
   - "Crawl" (variant="outline") — disabled when 0 stocks selected
   - "Analyze" (variant="outline") — disabled when 0 stocks selected
   - "Score" (variant="outline") — always enabled (operates on all tracked)
   - "Run Full Pipeline" (variant="default") — always enabled
4. On trigger: POST to relevant endpoint → receive `job_id`
5. Show toast "Operation started" with job type
6. Auto-switch to Jobs tab programmatically
7. Button shows Loader2 spinning icon while request is in-flight (not during background job)

### Job Monitor Flow (Jobs Tab)
1. Table loads via `GET /api/admin/jobs` with TanStack Query
2. Columns: Type (badge), Status (colored badge), Duration (calculated), Created
3. Duration: computed as `completed_at - started_at` (or "—" if not completed)
4. Polling: `refetchInterval: 3000` when any job has status `running` or `pending`
5. Polling auto-stops when all visible jobs are `completed` or `failed`
6. Click row to expand: shows `result` (JSON formatted) or `error` (text) from `GET /api/admin/jobs/{id}`
7. Expanded content: inside Collapsible (existing component), light muted background

### Loading States
- Initial page load: 8 Skeleton rows (height h-10, full width) per existing pattern
- Tab switch: instant (data cached by TanStack Query after first load)
- Mutation in-flight: Button shows Loader2 spinner, disabled state

### Error States
- API unreachable: ErrorState component with existing `common.loadError` / `common.connectionError` copy
- 409 conflict (admin lock): toast "An operation is already running. Wait for it to complete."

## Data Flow Contract

### API Types (to add to `lib/types.ts`)

```typescript
export interface TrackedStock {
  symbol: string;
  name: string | null;
  exchange: string | null;
  industry: string | null;
  is_tracked: boolean;
}

export interface TrackedStocksResponse {
  stocks: TrackedStock[];
  count: number;
}

export interface AdminJob {
  id: number;
  job_type: "crawl" | "analyze" | "score" | "report" | "pipeline";
  status: "pending" | "running" | "completed" | "failed";
  params: Record<string, unknown> | null;
  created_at: string | null;
  started_at: string | null;
  completed_at: string | null;
}

export interface AdminJobDetail extends AdminJob {
  result: Record<string, unknown> | null;
  error: string | null;
}

export interface AdminJobsResponse {
  jobs: AdminJob[];
  count: number;
}

export interface TriggerResponse {
  job_id: number;
  status: string;
  job_type: string;
  symbols?: string[];
  symbol?: string;
}
```

### TanStack Query Hooks (to add to `lib/queries.ts`)

| Hook | Type | Query Key | Endpoint | Options |
|------|------|-----------|----------|---------|
| `useTrackedStocks` | query | `["admin", "stocks"]` | `GET /api/admin/stocks` | `staleTime: 30s` |
| `useAdminJobs` | query | `["admin", "jobs"]` | `GET /api/admin/jobs` | `refetchInterval: 3000` when any job is running/pending, else `false` |
| `useAdminJobDetail` | query | `["admin", "jobs", id]` | `GET /api/admin/jobs/{id}` | `enabled: !!id` |
| `useAddStock` | mutation | invalidates `["admin", "stocks"]` | `POST /api/admin/stocks` | `onSuccess: toast + refetch` |
| `useRemoveStock` | mutation | invalidates `["admin", "stocks"]` | `DELETE /api/admin/stocks/{symbol}` | `onSuccess: toast + refetch` |
| `useTriggerCrawl` | mutation | invalidates `["admin", "jobs"]` | `POST /api/admin/crawl` | `onSuccess: toast + switch to Jobs tab` |
| `useTriggerAnalyze` | mutation | invalidates `["admin", "jobs"]` | `POST /api/admin/analyze` | `onSuccess: toast + switch to Jobs tab` |
