# Milestone v1.2: Admin Console

**Status:** ✅ SHIPPED 2026-04-23
**Phases:** 11-13 (+ 12.1 polish)
**Total Plans:** 8

## Overview

Admin Console milestone adds full administrative capabilities to LocalStock. Backend API endpoints for stock management, pipeline control, and job history. Frontend admin page with stock table, pipeline triggers, real-time job monitoring with toast notifications, and AI report generation with a dedicated job detail page.

## Phases

### Phase 11: Admin API Endpoints

**Goal**: Backend exposes REST endpoints for admin operations (stock CRUD, pipeline trigger, job status)
**Depends on**: Phase 5 (automation service), Phase 1 (pipeline)
**Requirements**: ADMIN-01, ADMIN-02, ADMIN-03, ADMIN-04
**Plans**: 2 plans

Plans:
- [x] 11-01-PLAN.md — Data layer foundation (models, migration, repositories)
- [x] 11-02-PLAN.md — Admin API endpoints + service + tests

**Success Criteria:**
1. POST /api/admin/stocks — add a stock symbol to the watch list
2. DELETE /api/admin/stocks/{symbol} — remove a stock from the watch list
3. POST /api/admin/crawl — trigger crawl for 1 or more symbols (returns job ID)
4. POST /api/admin/analyze — trigger analysis + scoring for 1 or more symbols
5. POST /api/admin/pipeline — trigger full daily pipeline
6. POST /api/admin/report — generate AI report for a specific symbol
7. GET /api/admin/jobs — list recent pipeline/job history with status
8. GET /api/admin/jobs/{id} — get detailed job status + errors

## Milestone Summary

**Decimal Phases:**
- Phase 12.1: Performance & Polish (inserted after Phase 12 for UX improvements)

**Key Decisions:**
- Used background job model (1 job per batch, not N jobs per symbol)
- Polling every 3s for real-time job updates (existing pattern, not SSE)
- Sheet/drawer replaced with dedicated job detail page per user preference
- shadcn base-nova preset with @base-ui/react primitives (not Radix)
- Cache invalidation via TanStack Query queryClient.invalidateQueries
- Job transition detection via useJobTransitions hook comparing snapshots

**Issues Resolved:**
- Job row highlight after toast action click (scrollIntoView + CSS animation)
- Cache staleness after operations (targeted invalidation per job type)
- Sheet reopen UX (replaced entirely with page navigation)
- Error heading mismatch for non-report jobs

**Issues Deferred:**
- UAT manual testing (29 scenarios across phases 7, 12, 13)
- Human verification for phases 9, 13

**Technical Debt Incurred:**
- shadcn Sheet component installed but unused after design pivot (may be useful later)
- No formal v1.2 requirements in REQUIREMENTS.md (phases driven by roadmap goals)
