## Milestone Summary

**Key Decisions:**

- No new Python dependencies — all v1.4 features use already-installed packages (pandas-ta, Ollama SDK, pydantic)
- No new API endpoints — content_json JSONB absorbs new StockReport fields automatically
- No new DB tables — signals route through existing indicator_data dict into LLM prompt
- Pure OHLC math for candlestick patterns — no TA-Lib C binary dependency needed
- MFI(14) for volume divergence — gated on avg_volume ≥ 100k shares/day
- num_ctx raised to 8192 (from 4096) — sufficient headroom for new prompt content
- Auto-correction for price inversions — swap rather than null when fixable

**Issues Resolved:**

- formatVND rounding dropped decimal precision for trade prices (Math.round removed)
- NaN values in financial DataFrames caused PostgreSQL JSON storage errors
- vnstock 4.0.1 changed DataFrame format from long to wide (detection + handling added)
- LLM sometimes omitted risk_rating field (prompt strengthened to REQUIRED)
- Price level inversions (stop > entry, entry > target) — auto-correction added

**Issues Deferred:**

- Nyquist compliance: 4 phases have draft VALIDATION.md files (not fully compliant)
- Missing VERIFICATION.md for Phases 18-19 (verified through downstream Phase 20)
- ROADMAP.md plan checkboxes for phases 19-21 were unchecked (cosmetic)
- Phase 21 VERIFICATION.md status: human_needed (UAT completed all visual checks)

**Technical Debt Incurred:**

- Nyquist validation framework partially adopted but not enforced
- vnstock 4.x wide-format financial data stored as single JSON blob (not parsed into individual rows)
- v1.3 code review warnings (3 items) carried forward
